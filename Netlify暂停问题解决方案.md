# 🚨 Netlify 网站暂停问题 - 解决方案

**问题**: Netlify 网站因达到使用限制被暂停  
**错误信息**: "Site not available - This site was paused as it reached its usage limits"  
**解决日期**: 2025-12-31

---

## 🔍 问题分析

### 可能的原因

1. **带宽限制**
   - 免费计划：100GB/月
   - 如果访问量大，可能超出限制

2. **构建次数限制**
   - 免费计划：300分钟/月
   - 频繁部署会消耗构建时间

3. **功能使用限制**
   - 某些高级功能需要付费

---

## ✅ 解决方案

### 方案一：升级 Netlify 计划（推荐）

**优点**:
- ✅ 立即恢复网站
- ✅ 更高的使用限制
- ✅ 更多功能
- ✅ 专业支持

**步骤**:

1. **登录 Netlify**
   ```
   访问: https://app.netlify.com
   使用您的账号登录
   ```

2. **进入账单页面**
   - 点击右上角头像
   - 选择 "Billing" 或 "Account settings"
   - 查看当前计划

3. **升级计划**
   - **Pro 计划**: $19/月
     - 带宽：400GB/月
     - 构建时间：1000分钟/月
     - 更多功能
   
   - **Business 计划**: $99/月
     - 带宽：1TB/月
     - 构建时间：5000分钟/月
     - 高级功能

4. **完成支付**
   - 选择支付方式
   - 完成支付
   - 网站自动恢复

---

### 方案二：优化使用量（免费计划）

**如果暂时不想付费，可以优化使用**:

1. **减少构建频率**
   - 只在重要更新时部署
   - 合并多个提交再部署
   - 使用 GitHub Actions 控制部署

2. **优化网站大小**
   - 压缩图片
   - 减少 JavaScript 文件大小
   - 使用 CDN 缓存

3. **等待下个月重置**
   - 免费计划每月重置使用量
   - 等待下个月自动恢复

---

### 方案三：迁移到其他平台（备选）

#### 选项1：Vercel（推荐）⭐

**优点**:
- ✅ 免费计划更慷慨
- ✅ 自动部署
- ✅ 全球 CDN
- ✅ 与 GitHub 完美集成

**步骤**:

1. **注册 Vercel**
   ```
   访问: https://vercel.com
   使用 GitHub 账号登录
   ```

2. **导入项目**
   - 点击 "New Project"
   - 选择 GitHub 仓库
   - 自动检测配置

3. **部署设置**
   ```json
   {
     "buildCommand": "npm run build",
     "outputDirectory": "dist",
     "framework": "vite"
   }
   ```

4. **部署**
   - 点击 "Deploy"
   - 等待部署完成
   - 获得新的网站地址

**Vercel 免费计划限制**:
- 带宽：100GB/月
- 构建时间：6000分钟/月
- 更适合个人项目

---

#### 选项2：GitHub Pages

**优点**:
- ✅ 完全免费
- ✅ 与 GitHub 集成
- ✅ 无带宽限制（合理使用）

**缺点**:
- ❌ 不支持服务端功能
- ❌ 构建需要 GitHub Actions

**步骤**:

1. **启用 GitHub Pages**
   - 进入仓库 Settings
   - 找到 "Pages"
   - 选择分支和目录

2. **配置 GitHub Actions**
   - 创建 `.github/workflows/deploy.yml`
   - 自动构建和部署

3. **更新域名**
   - 在仓库设置中配置自定义域名
   - 更新 DNS 记录

---

#### 选项3：Cloudflare Pages

**优点**:
- ✅ 免费计划很慷慨
- ✅ 全球 CDN
- ✅ 自动部署
- ✅ 与 GitHub 集成

**步骤**:

1. **注册 Cloudflare**
   ```
   访问: https://pages.cloudflare.com
   使用账号登录
   ```

2. **连接 GitHub**
   - 授权 Cloudflare 访问仓库
   - 选择仓库

3. **配置构建**
   ```
   Build command: npm run build
   Build output directory: dist
   ```

4. **部署**
   - 点击 "Save and Deploy"
   - 等待部署完成

## 🔄 迁移到 Vercel（详细步骤）

### 如果选择迁移到 Vercel

1. **注册 Vercel**
   ```
   访问: https://vercel.com/signup
   使用 GitHub 账号登录
   ```

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 选择 GitHub 仓库
   - 选择 "shoe-store-management" 仓库

3. **配置项目**
   ```
   Framework Preset: Vite
   Root Directory: ./
   Build Command: npm run build
   Output Directory: dist
   Install Command: npm install --legacy-peer-deps
   ```

4. **环境变量**（如果有）
   - 添加 Supabase 相关环境变量
   - 添加其他必要的配置

5. **部署**
   - 点击 "Deploy"
   - 等待构建完成
   - 获得新的网站地址

6. **自定义域名**（可选）
   - 在项目设置中添加域名
   - 更新 DNS 记录
   - 等待 SSL 证书生成

7. **更新 GitHub**
   - 更新 README 中的部署地址
   - 更新文档中的链接
